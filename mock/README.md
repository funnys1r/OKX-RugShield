# Mock 测试包

这个目录用于比赛演示、评委复现和本地联调。

## 文件说明

### `mock-rug-event.json`
一条模拟的高危 Rug 事件，包含：
- 模拟 Threat Report
- 模拟流动性深度变化
- 模拟多钱包持仓暴露
- 动态最优退出建议

## 设计目的

评委在评审时，不可能真的去买一个即将归零的土狗来验证系统。
因此 RugShield 提供一个可复现的 mock 事件输入，用来演示：

1. Scout 识别高危事件
2. Guardian 检查多钱包暴露
3. Guardian 结合流动性深度与滑点阈值，计算动态最优退出比例
4. Guardian 输出模拟防守日志

## 推荐用法

### 方式 1：配合 `npm run demo`
作为讲解素材使用，向评委解释当前 mock demo 对应的事件语义。

### 方式 2：未来接入更完整测试流
后续可扩展为：
- 将 `mock-rug-event.json` 注入 CLI
- 或在 OpenClaw 测试环境中作为一轮标准输入
- 回放一整次 Scout → Guardian 联动流程

## 注意

当前 mock 测试包是**演示输入样例**，不是完整自动回放框架。
它的作用是让评委和开发者清楚看到：
**RugShield 不只是口头描述风险，而是已经在仓库中提供了可复现的测试事件载体。**
