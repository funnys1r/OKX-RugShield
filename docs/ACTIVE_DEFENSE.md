# OKX RugShield 🛡️ | 主动防守说明

这份文档说明 RugShield 在设计上如何从“交易时被动检测”推进到“持续巡检 + 主动防守”。

## 当前原型层

当前仓库已提供的主动性主要体现在：

- Scout 负责持续巡检/侦察的角色定义
- Guardian 负责在检测到高风险信号后立即检查持仓暴露
- mock 演示中提供了从风险发现到策略生成的完整闭环

## 当前代码层新增支撑

当前仓库中的主动防守相关代码入口包括：

- `cli/replay-mock.js`：读取 mock 事件并回放 Scout → Guardian 联动
- `lib/exit-strategy.js`：根据风险分数、流动性深度和滑点阈值计算动态退出比例

## 下一阶段方向

下一阶段若继续增强，可进一步实现：

- 真正的周期性巡检任务
- Pending Transaction / Mempool 监听
- 主动通知用户撤退或直接交给 Guardian 生成方案
- 与 AA 钱包 / 账户抽象执行策略层进一步结合
