# OKX RugShield 🛡️ | 主动巡检原型

这份文档说明当前仓库中用于体现“主动性”的原型能力。

## 当前原型目标

评审对创新性的核心关注点之一，是项目是否只是“用户发起交易时被动检测”，还是具备更强的主动性。

当前仓库提供的主动巡检原型，目标是展示：
- Scout 可以主动发现高风险事件
- Guardian 可以在没有等待用户先发起交易的情况下，先检查暴露并生成防守建议

## 当前代码入口

```bash
npm run patrol:mock
```

该命令会：
1. 读取 `mock/mock-rug-event.json`
2. 以“主动巡检”的口径回放一轮 Scout → Guardian 联动
3. 输出主动告警、暴露检查与动态退出建议

## 当前能力边界

当前原型仍是：
- mock 输入驱动
- 原型级主动巡检演示
- 非真实后台守护进程

也就是说，这一能力当前用于证明：
**RugShield 的设计目标不是只做交易时被动检测，而是在往主动防守 Agent 的方向演进。**
